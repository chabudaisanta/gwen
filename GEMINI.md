# gwen プロジェクト指示書 (GEMINI.md)

このファイルは、AIエージェント（Antigravity）が本プロジェクト「gwen」で動作する際の開発・行動指針を定義したものです。

---

## 1. 開発方針・ディレクトリ構成ルール
* **ヘッダーオンリー**: 本ライブラリはヘッダーオンリーです。実装はすべて `include/gwen/` 配下の適切な名前空間ディレクトリに `.hpp` 形式で作成・追加します。
* **C++23 Concept の利用**: グラフ (Graph) や代数構造 (Monoid, Abel) などの汎用コンポーネントは、C++23 `concept` を用いて厳密な制約を設けます。特殊なものは都度検討します。
* **外部参照の隔離**: 外部のリファレンスコード（例: `nyaan` 等）は、自作コードと混ざらないように `references/` ディレクトリ配下に隔離します。
* **ビルド成果物の除外**: ビルド成果物やデバッグ用テンポラリファイルはルートを汚さないよう `.build/` ディレクトリへ出力し、Git追跡から完全に除外（`.gitignore`）します。
* **エディタ設定の除外**: `.vscode/` 設定フォルダはローカルのみで保持し、Gitには含めません。

---

## 2. ドキュメントとスクリプトの形式的管理
* **ドキュメント管理**: `README.md`, `GEMINI.md`, `planning/TODO.md` は形式的に管理します。プロジェクトの状態や規約が変更された場合は、これらのドキュメントを常に最新の状態に保つようにします。
* **スクリプト管理**: `scripts/` に配置するスクリプトは、用途を明確にし、必要に応じて `README.md` 等に使い方を明記して形式的に管理します。すべてターミナル上で完結するようにします。

---

## 3. テスト・検証ルール
* **新規実装時の必須対応**: 新規機能追加やモジュール移植時は、必ず「Google Test による単体テスト」と「competitive-verifier 用の verify コード」の両方をセットで作成し、パスさせます。
* **Google Testの利用**: テストフレームワークには C++ Google Test を使用し、ビルドには CMake を使用します。ユニットテストは `test/unit/` 配下に各モジュール用の gtest コードを作成します。
* **Verifyの実行**: オンラインジャッジ提出用のコードは `verify/` 配下に格納し、`competitive-verifier` を用いて検証を行います。

---

## 4. コミュニケーションと言語ルール
* **言語仕様**: ユーザーへの最終的な回答、コメント、ドキュメントの更新はすべて自然な **日本語** で行ってください。
* **実装計画の優先**: 大きなリファクタリングや機能追加を行う前には、必ず `implementation_plan.md` を作成してユーザーの承認を得るようにプロセスを守ってください。

## 5. コミットメッセージのルール
本プロジェクトでは以下のプレフィックスを付けた緩やかな Conventional Commits を使用してください。
- \eat:\ : 新機能・新しいモジュール
- \ix:\ : バグ修正
- \docs:\ : ドキュメントの更新
- \	est:\ : テスト・検証コードの追加
- \chore:\ : CI/CD、その他雑務
