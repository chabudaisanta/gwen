# 移植タスク: mod/mod61.hpp

## 1. 概要
`master` ブランチから `mod/mod61.hpp` を `rebuild-v2` ブランチへ移植します。
このモジュールは、$2^{61}-1$ を法とする高速なモジュラ演算（主にローリングハッシュで使用される）を提供します。

## 2. 仕様案

### 2.1 パス
- `include/gwen/mod/mod61.hpp`

### 2.2 Concept とクラス化
- 旧実装では `namespace gwen::rhash` 内に `add_mod`, `mul_mod` 等の constexpr 関数群として実装されていました。
- 本タスクでは、新設した `modint` コンセプトに適合するように、新たに **`ModInt61`** クラス（パスカルケース規則に基づく）としてラップして実装することを提案します。
- `ModInt61` は内部に `u64` 型の値を保持し、`+`, `-`, `*`, `/`, `+=`, `-=`, `*=`, `/=` などの演算子オーバーロードを提供します（除算は内部で `* inv()` を行います）。
- `val()`, `inv()`, `pow(n)` および `static constexpr u64 mod()` を実装し、`modint` コンセプトを完全に満たします。また、等値比較 `==`, `!=` も実装します。
- すべてのメソッド・演算子に `constexpr` を付与し、コンパイル時計算を可能にします。
- これにより、他のモジュラ演算と同様に一貫したインターフェースでローリングハッシュや行列累乗等のアルゴリズムに組み込むことができます。

### 2.3 詳細仕様
- **定数**: `mod61 = (1ull << 61) - 1`
- **乗算・オーバーフロー対策**: `u128` で計算後、ビットシフトと加算により高速な剰余算 (`res = (t >> 61) + (t & msk61)`) を行います。計算結果が $2^{61}-1$ 以上になるケースを防ぐため、最後に `if (res >= mod61) res -= mod61;` の正規化を必ず行います。
- **コンストラクタ**: 整数から初期化する際、負の値が渡された場合も適切に法を足し込んで正の値（$0 \le x < 2^{61}-1$）に丸めます。
- **逆元 `inv()`**: フェルマーの小定理より `pow(mod61 - 2)` にて算出します。値が0の場合は `assert` で弾きます。
- 戻り値の変数名は `res` に統一します。
